DevOps - Prashanth

DevOps is a cultural and technical approach that unifies software development (Dev) and IT operations (Ops) to enhance collaboration, streamline processes, and deliver software faster and more reliably. It focuses on breaking down silos between teams, enabling continuous integration, delivery, and deployment of software.

Key principles of DevOps include automation, collaboration, and continuous feedback. Automation reduces repetitive tasks like testing and deployment, ensuring efficiency and consistency. Collaboration fosters open communication between developers, operations teams, and stakeholders, aligning goals and improving problem-solving. Continuous feedback ensures that products evolve based on user needs and performance metrics.

DevOps practices rely on tools for source control (e.g., Git), CI/CD pipelines (e.g., Jenkins, GitLab CI), infrastructure as code (e.g., Terraform), and monitoring (e.g., Prometheus, Grafana). These tools enable rapid iterations, scalability, and resilience in the development lifecycle.

The benefits of DevOps include faster time to market, improved quality, and enhanced system reliability. Organizations adopting DevOps often experience better team morale and reduced downtime, as issues are detected and resolved proactively.

By integrating development and operations with a culture of shared responsibility and automation, DevOps accelerates innovation while maintaining stability and scalability.

Redirection and Piping
  • Redirection:
    • Standard input redirection: 0 <
    • Standard output redirection: 1 > >>
    • Error redirection: 2 > >>
    • Redirect Linux command output to a file
    • Example: date > date.txt, adds the date to the date.txt file
    • Example: echo "hi welcome" >> date.txt, appends a new line to date.txt
    • Error redirection example: data 2>> date.txt, captures errors
    • Use semicolons to add multiple commands in one line and redirect their output
    • Examples:
      • (date; data; ls; pwd) > date.txt: redirects standard output, skips errors
      • (date; data; ls; pwd) &> date.txt: redirects both standard output and errors
  • Piping:
    • Output of one command is input for another
    • Example: cat -n devops.txt | grep red, searches for "red" in devops.txt
    • Case-insensitive search: grep -i
    • Real-time example: docker ps -a | grep -i sarath > sarath_containers.txt
  • History:
    • Lists previously executed commands
    • Example: !10 runs command at line 10
  • File operations:
    • Last 5 lines: cat -n date.txt | tail -5
    • First 5 lines: cat -n date.txt | head -5
    • Lines 21-30: cat -n devops.txt | head -30 | tail -10

Wild Card Characters
  • *, ?, [], [!], {}

    • ? - exactly one character

Cron Jobs and Shell Scripting
  • Cron Jobs: Schedule jobs at a particular time in Linux
    • Syntax to set cron jobs: minutes hours dayofthemonth month dayoftheweek /
    • For example: 30 20 10 12 2 - "At 20:30 on day-of-month 10 and on Tuesday in December."
    • Use crontab.guru for easy cron expression generation
    • To edit cron jobs: crontab -e
    • Examples:
      • Daily: 30 20 * * * - runs every day at 20:30
      • Every Tuesday: 30 20 * * 2 - runs every Tuesday at 20:30
      • 1st of every month: 30 20 1 * * - runs on the 1st of every month at 20:30
      • Every hour: 30 * * * * - runs every hour at minute 30
      • Every minute: * * * * * - runs every minute
  • Shell Scripting:
    • File extension: .sh or .bashrc
    • Start script with shebang: #!/bin/bash - defines the shell interpreter for the script
    • List available shell interpreters: cat /etc/shells
    • Document the script properly
    • Make the script executable: chmod 777 filename.sh
    • Execute the script: ./filename.sh
    • Enable debugging: set -x (shows command and output)
    • Disable debugging: set +x
    • Enable brace expansion: set -B (e.g., mkdir sss{1..232})
    • Disable brace expansion: set +B
    • Multiple comments in the script:
      • Use << COMMENT ... COMMENT to comment multiple lines
    • Write interactive shell scripts

Package Management, User Creation, and System Management
  • Installing Packages in Linux
    • Package managers for different Linux distributions:
      • Ubuntu/Debian/Kali: apt or apt-get update
      • CentOS/RedHat/AWS: yum
      • Alpine: apk
      • Others: rpm, dmg, etc.
    • Updates to package managers can only be done by the root user. Use sudo before trying to install or update any package.
    • Useful commands:
      • List all available packages: apt list
      • Search for a package: apt list | grep -i apache2
      • List installed packages: apt list --installed
      • Search for a file: sudo find / -name tomcat10
      • Uninstall a package: sudo apt remove tomcat10
      • Remove dependencies after uninstalling: sudo apt-get purge tomcat10
      • Remove unused packages: sudo apt autoremove
      • Clean package cache: sudo apt autoclean
    • To completely uninstall a package and its dependencies:
      • sudo apt remove tomcat10
      • sudo apt-get purge tomcat10
      • sudo apt autoremove
      • sudo apt autoclean
  • Creating Users in Linux
    • Each user in Linux is assigned a User ID (UID):
      • UID 0-1000: System-generated users
      • UID > 1000: User-created users
    • Example user entry in /etc/passwd: 
      root:x:0:0:root:/root:/bin/bash
      • root: Username
      • x: Placeholder for password
      • 0: UID
      • 0: GID
      • root: Comment
      • /root: Home directory
      • /bin/bash: Shell type
    • List all users: cat /etc/passwd
    • Example to create a user: sudo useradd madhu (creates default user with UID 1001)
    • To specify user options during creation: 
      sudo useradd -s /bin/bash -m -d /home/prakash -c "developer" prakash
      • -s: Set shell type
      • -m: Create home directory
      • -d: Set home directory
      • -c: Add comment
    • Specify custom UID: sudo useradd -s /bin/bash -m -d /home/nanna -c "nanna" -u 9999 nanna
    • To check a specific user: cat /etc/passwd | grep -i nanna
    • To view all options available for useradd: man useradd
  • Shell Check: Verify if a shell is valid: 
    grep -qx /bin/sh /etc/shells
    • -x: Ensures the entire line matches
    • -q: Suppresses output, used only for checking

  • Networking:
    • To connect the servers and share the resources and enable communication between resources.
    • Example: Dedicating a common printer to all employees in an organization instead of setting up each employee with a printer (which can reduce cost of infrastructure).
    • Intranet and Internet
    • In Intranet - IP address will be unique within the organization, and might not be unique compared to other organizations.
    • In Internet - IP address are always unique.
  • IP Address:
    • Sample: 23.34.45.56
    • IPv4 - 32 bits in size - 4 octets - each octet is 8 bits
    • The value of an IP address should range from (0-255).(0-255).(0-255).(0-255)
    • 23.306.55.1221 - Invalid IP address
    • 23.232.32.133 - Valid IP address
    • We can generate up to 232 IP addresses with above combinations - approx 4 billion IP addresses.
    • IPv6 - 128 bit - 8 octets - each octet 16 bit in size
    • Sample: aaa.bbb.ccc.ddd.eee.fff.ggg.hhh
    • (0-65536).(0-65536).(0-65536).(0-65536).(0-65536).(0-65536).(0-65536).(0-65536)
    • IPv6 can accommodate 2128 (340,282,366,920,938,463,463,374,607,431,768,211,456) number of IP addresses.
    • IPv6 ensures we will never run out of available IP addresses.
    • Current use cases: mobiles, IoT devices, car sensors, CCTV
  • IP Address Classes:
    • Class A: 0-126.0-255.0-255.0-255
    • Class B: 128-191.0-255.0-255.0-255
    • Class C: 192-223.0-255.0-255.0-255
    • Class D and E: 224-255.0-255.0-255.0-255
  • Private IP Ranges:
    • Class A: 10.0.0.0 to 10.255.255.255
    • Class B: 172.31.0.0 to 172.31.255.255
    • Class C: 192.168.0.0 to 192.168.255.255
  • ISP Role: When private organizations with the same IP address merge, ISP ensures unique addresses for both.
