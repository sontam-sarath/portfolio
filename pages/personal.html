How I Work & What I Value

Reliability

I show up, follow through, and write the kind of code I'd be happy to debug at 2am. Boring reliability is deeply underrated in software teams.

Clarity

Good engineering is mostly good communication. I write code, comments, and docs that don't require a translator to understand a week later.

Ownership

I care about outcomes, not just output. If something I built is causing pain, I want to know and I want to fix it ‚Äî not wait for a ticket.
